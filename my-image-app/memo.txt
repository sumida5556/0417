# 画像アップロードアプリ

Next.js + Vercel Blob + Neon PostgreSQL を使った画像アップロードアプリケーション

## 技術スタック

- **フロントエンド**: Next.js 16.2.4 (App Router), React, Tailwind CSS
- **バックエンド**: Next.js API Routes
- **データベース**: Neon PostgreSQL
- **ORM**: Prisma 5.22.0
- **ストレージ**: Vercel Blob
- **デプロイ**: Vercel

## セットアップ手順

### 1. リポジトリのクローン

```bash
git clone <your-repo-url>
cd my-image-app
```

### 2. 依存関係のインストール

```bash
npm install
```

### 3. 環境変数の設定

`.env` ファイルを作成：

```env
DATABASE_URL="postgresql://username:password@host/database?sslmode=require"
```

**DATABASE_URLの取得方法:**
1. [neon.tech](https://neon.tech) でアカウント作成
2. 新規プロジェクト作成
3. Connection String をコピー

### 4. データベースのマイグレーション

```bash
npx prisma generate
npx prisma migrate dev --name init
```

### 5. ローカル開発サーバーの起動

```bash
npm run dev
```

http://localhost:3000 でアクセス

### 6. Vercelへのデプロイ

```bash
npm install -g vercel
vercel login
vercel --prod
```

**デプロイ後の設定:**

1. **Vercel Blob ストレージの作成**
   - Vercel Dashboard → Storage → Create Database → Blob
   - プロジェクトに接続

2. **環境変数の設定**
   - Settings → Environment Variables
   - `DATABASE_URL` を追加（Production, Preview, Development）
   - `BLOB_READ_WRITE_TOKEN` は自動設定される

3. **再デプロイ**
   - Deployments → 最新のデプロイ → Redeploy
